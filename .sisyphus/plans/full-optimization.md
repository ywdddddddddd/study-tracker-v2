## Phase 1: 自动保存机制

### 目标
所有模块在切换日期/切换 tab/离开页面时自动保存，不需要手动点按钮

### 新增文件
- `src/hooks/useAutoSave.ts` — 通用自动保存 Hook
- `src/hooks/useDirty.ts` — 脏数据追踪 Hook
- `src/components/ui/SaveIndicator.tsx` — 保存状态指示器组件

### useAutoSave 设计
```typescript
function useAutoSave<T>({
  data: T,              // 当前数据
  saveFn: (data: T) => Promise<void>,  // 保存函数
  key: string,          // 唯一标识(如日期)
  enabled: boolean,     // 是否启用
}): {
  isSaving: boolean;    // 保存中
  isDirty: boolean;     // 有未保存变更
  lastSaved: Date|null; // 最后保存时间
  save: () => Promise<void>;  // 手动触发
  saveStatus: 'idle' | 'saving' | 'saved' | 'error';
}
```

### 各模块修改
- **DailyPlan**: 日期切换时保存 → 页面离开时保存 → Tab切换时保存
- **WeeklyReview**: 同上
- **Fitness**: 同上 + 加练数据
- **Nutrition**: 食物条目增删改即刻保存（已有近似行为）
- **Health**: 体重/睡眠增删改即刻保存，档案切换时保存

### 离开拦截
```typescript
// App.tsx 中拦截 tab 切换
// 如果当前模块有脏数据，先保存再切换
onTabChange = async (newTab) => {
  await currentModuleSave();
  setActiveTab(newTab);
}
```

### 视觉反馈
- 保存按钮改为状态指示器：`● 已修改` → `◌ 保存中...` → `✓ 已保存` → `✗ 保存失败`
- 全局底部状态栏：网络状态 + 同步状态
- 加载数据时显示骨架屏/Spinner

## Phase 3: AI 多 Agent 工作流引擎

### 目标
链式多 Agent 分析 + 校验 + 结构化 JSON 输出，替换当前的单一 system prompt

### 工作流
```
触发 → Agent-A(饮食分析) ──→ 中间结果 JSON
     → Agent-B(健身分析) ──→ 累积中间结果 JSON
     → Agent-C(学习分析) ──→ 累积中间结果 JSON
     → Agent-D(综合安排) ──→ 完整安排 JSON
     → Validator(校验+评分) ──┐
         ↓ pass (>80分)       ↓ fail (≤80分)
       显示仪表盘         → Agent-D(重试, 带校验反馈)
                              ↓
                           Validator(再校验)
                              ↓ (最多3次)
                         降级显示 + 警告标记
```

### 新增文件
- `src/lib/ai/types.ts` — AI 相关的 TypeScript 类型和 JSON Schema
- `src/lib/ai/providers.ts` — LLM Provider 管理器
- `src/lib/ai/agents.ts` — Agent 定义（system prompts + 参数）
- `src/lib/ai/orchestrator.ts` — 多 Agent 编排引擎
- `src/lib/ai/formatter.ts` — 格式整理 Agent（Model B）
- `src/lib/ai/validator.ts` — 校验 Agent + 评分逻辑
- `src/lib/ai/settings-store.ts` — LLM 设置本地存储

## 风险点

1. **Supabase 速率限制**: 免费版 500 req/s，多 agent 并发 + 同步可能触达
   - 缓解：合并请求、批量同步、请求队列

2. **API Key 安全**: LLM API Key 存储在浏览器 localStorage
   - 缓解：明确提示用户"仅存储在本地浏览器"，提供加密选项

3. **多 Agent 耗时**: 6 个 agent 链式调用可能耗时 30-60s
   - 缓解：步骤链实时展示进度、支持中止、超时处理

4. **离线数据冲突**: 多设备同时离线再同步
   - 缓解：最后写入胜出 + updated_at 时间戳比较
